# AGENTS.md

## Язык и стиль
- Всегда отвечать на русском языке.
- Писать коротко и по делу.
- Комментарии в коде писать простым человеческим языком.

## Код и качество
- Запрещено использовать `eslint-disable`.
- Запрещено менять конфигурацию линтера без явного запроса.
- Перед подготовкой к деплою или коммиту обязательно запускать `npm run precommit`.
- После внесения изменений обязательно запускать приложение локально и проверять логи.
- Все багфиксы покрывать тестами позитивных сценариев.
- Новые типы/DTO/интерфейсы выносить в отдельные файлы (`*.interfaces.ts`, `*.dto.ts`).
- При индексации/поиске игнорировать служебные и сгенерированные папки (`build`, `target`, `.idea`, `.gradle`, `dist`, `node_modules`).

## Внешние API
- Перед интеграцией с внешним API сначала сделать smoke-check через `curl`/эквивалент.
- Если нужны ключи/URL — запросить у пользователя.
- Приоритет на бесплатные API/тарифы.
- Если действие затрагивает внешние сервисы или сеть, сначала явно согласовать это с пользователем.

## Git workflow
- Не делать `git commit` и `git push` без явного запроса пользователя.
- Не использовать разрушительные git-команды без подтверждения.

## Документация и релизы
- После доработок обновлять README по реализованному функционалу.
- При каждом деплое публиковать release notes с версией приложения.

## Ограничения окружения
- Если Docker недоступен, остановиться и сообщить пользователю.
- Если проект не собирается из-за недоступности зависимостей/артефактов, остановиться и сообщить пользователю.
